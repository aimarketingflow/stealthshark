1 Prerequisites & Requirements

What you need before installing StealthShark

System Requirements

  • macOS 10.13+ (High Sierra or later) for the standalone app
  • macOS or Linux for the source install
  • 50 MB free disk space for the app (+ space for captures)
  • Admin access for packet capture permissions

For Source Install Only

  • Python 3.9+ (3.12–3.14 recommended)
  • pip (comes with Python)
  • tcpdump (pre-installed on macOS and most Linux)
  • git (to clone the repo)
Don't have Python?

macOS: Install via Homebrew: brew install python3
Linux: sudo apt install python3 python3-pip (Debian/Ubuntu) or sudo dnf install python3 python3-pip (Fedora)

Run StealthShark as a background daemon on servers or headless systems. No GUI required.

4 Granting Capture Permissions

Packet capture requires elevated permissions on most interfaces

StealthShark uses tcpdump to capture packets. Most interfaces (Ethernet, VPN, Wi-Fi) require root/sudo access. Loopback (lo0) works without sudo.

Option A: Run with Sudo (Simplest)

Terminal
# Source install
sudo python3 multi_interface_shark_gui.py

# Headless daemon
sudo python3 persistent_wireshark_monitor.py

Option B: Install ChmodBPF (macOS — Recommended)

If you have Wireshark installed, it includes ChmodBPF which grants capture permissions without sudo:

Terminal
# Check if ChmodBPF is installed (from Wireshark)
ls /Library/LaunchDaemons/org.wireshark.ChmodBPF.plist

# If not installed, install Wireshark from https://www.wireshark.org
# During install, check "Install ChmodBPF"

Option C: Configure Sudoers (Advanced)

Terminal
# Allow tcpdump without password prompt
sudo visudo

# Add this line (replace YOUR_USERNAME):
YOUR_USERNAME ALL=(ALL) NOPASSWD: /usr/sbin/tcpdump
Security Note

Granting NOPASSWD access to tcpdump allows any process running as your user to capture packets. Only use this on trusted machines.

StealthShark automatically tries methods in order: direct tcpdump → sudo tcpdump → fallback. It will prompt for your password if needed.

5 Verify It's Working

Confirm captures are being saved correctly

After launching and granting permissions, verify everything is working:

Check the GUI

  • Interface list shows discovered interfaces with UP/DOWN status
  • Status bar shows "Activity detected on: en0, lo0, ..." messages
  • Alert log shows "Started packet capture on [interface]" entries

Check Capture Files

Terminal
# List active capture files
find pcap_captures/ -name "*.pcap.gz" -exec ls -lh {} \;

# Expected output (files should be growing):
-rw-r--r--  15M  pcap_captures/session_.../ethernet/...-ch-en0.pcap.gz
-rw-r--r--  70M  pcap_captures/session_.../loopback/...-ch-loopback.pcap.gz

Check Running Processes

Terminal
# Verify tcpdump processes are running
ps aux | grep tcpdump | grep -v grep

# You should see one tcpdump per captured interface
You're all set!

If you see .pcap.gz files growing in size and tcpdump processes running, StealthShark is capturing successfully. Files are compressed in real-time—expect ~84% smaller than raw pcap.

6 Configuration & Settings

Customize capture duration, directories, and behavior

GUI Settings

The GUI provides controls for:

  • Capture Directory — where pcap files are saved (default: ./pcap_captures)
  • Capture Duration — how long each capture session runs (30s to 5 hours)
  • Auto-save — session state saved every 30 seconds for crash recovery

Command-Line Options (Headless)

Terminal
python3 persistent_wireshark_monitor.py \
    --capture-dir /path/to/captures \
    --duration 3600 \
    --interval 5
Flag Default Description
--capture-dir ./pcap_captures Directory for pcap output
--duration 3600 Capture duration in seconds (30–18000)
--interval 5 Seconds between activity checks

7 Understanding Your Capture Files

How captures are organized and how to use them

Directory Structure

File Tree
pcap_captures/
  # Active session
  session_20260211_113028/
    ethernet/
      20260211_113028-ch-en0.pcap.gz       # Ethernet capture
    vpn/
      20260211_113028-ch-utun0.pcap.gz     # VPN tunnel
    loopback/
      20260211_113028-ch-loopback.pcap.gz  # Localhost traffic
    firewall/
      20260211_113028-ch-pflog0.pcap.gz    # Firewall log
    other/
      20260211_113028-ch-ipsec0.pcap.gz    # IPSec

  # Archived sessions (auto-compressed after 1 hour)
  session_20260210_090000.tar.gz
  session_20260209_140000.tar.gz

  # Logs
  logs/
    wireshark_monitor_20260211.log

Opening Captures in Wireshark

Wireshark natively supports .pcap.gz files. No decompression needed:

Terminal
# Open directly in Wireshark
open -a Wireshark pcap_captures/session_.../ethernet/*.pcap.gz

# Or drag-and-drop the .pcap.gz file onto the Wireshark window

# To extract an archived session
tar -xzf pcap_captures/session_20260210_090000.tar.gz

Automatic Maintenance

  • Auto-archive: Sessions older than 1 hour are compressed to .tar.gz
  • Auto-cleanup: Archives older than 7 days are automatically deleted
  • Crash recovery: Session state saved every 30 seconds, restored on relaunch

Your captures are safe

Updating never touches your pcap_captures/ directory. All existing captures, archives, and session data are preserved.

9 Troubleshooting

Common issues and how to fix them

"Permission denied" or empty pcap files

tcpdump needs elevated permissions. Run with sudo or install ChmodBPF (see Step 4).

macOS blocks the app: "unidentified developer"

Go to System Settings → Privacy & Security, scroll down, and click "Open Anyway" next to the StealthShark message.

GUI doesn't appear (source install)

Make sure PyQt6 is installed: pip install PyQt6. On Linux, you may also need: sudo apt install libxcb-xinerama0.

App seems stuck or unresponsive

Kill and restart:
pkill -f multi_interface_shark_gui
Then relaunch. Session state will be restored automatically.

Captures are 0 bytes

This usually means the interface has no traffic, or tcpdump doesn't have permission on that interface. Check with: sudo tcpdump -i en0 -c 5

Disk filling up

StealthShark auto-archives sessions after 1 hour and deletes archives after 7 days. For high-traffic networks, you can reduce capture duration in the GUI settings or use --duration 1800 (30 min) for the daemon.
